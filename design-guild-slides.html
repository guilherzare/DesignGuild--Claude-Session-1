App vs Terminal

Deux façons d'utiliser
Claude Code

🖥

Application Desktop

Interface graphique pour démarrer rapidement. Gestion visuelle des projets. Idéale pour les débutants.

⌨️

Terminal (CLI)

Puissance maximale. Intégré dans ton workflow existant. Contrôle total sur le contexte et les fichiers.

COMMANDES CLI ESSENTIELLES

claude Lancer Claude Code dans le terminal
claude --continue Reprendre la dernière session
claude --resume Choisir une session passée
/help Afficher les commandes disponibles
/clear Vider l'historique de la conversation — Claude repart de zéro, sans mémoire des échanges précédents
Esc Esc Effacer l'entrée en cours
Base terminal
cd [dossier] Se placer dans un dossier
cd .. Remonter d'un niveau dans les dossiers
ls Lister les fichiers du dossier actuel
pwd Afficher le chemin du dossier actuel
code . Ouvrir le dossier actuel dans VS Code

CLAUDE.md

.MD le brief de Claude

Un simple fichier texte dans votre projet — Claude le lit à chaque session

✍️

Vous écrivez en français

Pas de code. Juste des phrases. "Réponds toujours en français." "Explique chaque modification."

🧠

Claude s'en souvient

À chaque nouvelle session, il relit ce fichier. Son comportement reste cohérent d'une fois à l'autre.

🎯

Adapté à votre équipe

Conventions de nommage, ton attendu, règles du projet. Claude s'adapte à votre contexte.

Repository

Local vs Git

💾

Repository Local

Ton projet existe uniquement sur ta machine. Parfait pour commencer à développer avant de partager.

☁️

Git (Remote)

Ton code est synchronisé sur un serveur distant (GitHub/GitLab). Collaboration en équipe possible.

→ git clone <url-du-repo> # Cloner un repo existant
→ git status # Voir les fichiers modifiés

AM
Anh-Minh VU
Git

Branches & Versions

🏠

Main

La branche principale, stable et partagée. On ne code jamais directement dessus.

🌿

Branch

Copie isolée du code pour travailler sans impacter main.

📸

Commit

Snapshot de tes modifications avec un message descriptif.

⬇️

Pull

Récupérer les derniers commits distants pour mettre à jour ta copie locale.

⬆️

Push

Envoyer tes commits locaux vers le serveur distant.

🔀

Merge / Pull Request

Intégrer ta branche dans main après review.

Step by step

Les étapes du workshop

  • 01 → Lancer Claude depuis le Desktop
  • 02 →
    Cloner le projet
    Claude "Clone le repo user-settings depuis GitHub" CLI git clone https://github.com/vuanh-minh/user-settings.git
  • 03 →
    Se déplacer dans le repo
    Claude ! cd user-settings CLI cd user-settings
  • 04 →
    Lancer le serveur → localhost:3000
    Claude "Start local server" CLI npm run dev
  • 05 →
    Créer sa branche
    Claude "Crée une branche profile-customization" CLI git checkout -b profile-customization
  • 06 →
    Itérer avec Claude
    Claude "Rajoute une section Customization (surname, avatar…)"
  • 07 →
    Commit & Push
    Claude "Commit et push mes changements" CLI git commit -m "feat: add customization section" && git push
  • 08 →
    Ouvrir une Pull Request
    Claude "Ouvre une PR sur GitHub" CLI gh pr create
